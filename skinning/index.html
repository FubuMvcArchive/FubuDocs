Skinning FubuDocs

FubuDocs can be customized by both CSS and your own layouts to completely control the look and feel of your documentation.

The custom style theme and overriden _chrome_ described below is found in the _host_ project if there is more than one FubuDocs project in a website.

CSS Theme's

You can very happily replace the entire style of the site by simply adding a stylesheet at /content/styles/fubudocs.theme.css. Do remember that FubuDocs uses Twitter Bootstrap for its basic styling and layout.

Custom Chrome

The basic layout of the site can be altered in any fashion if any of these files can be found in the host project. Do note that some of these files are named differently than the defaults in FubuDocs for, um, historical reasons that I don't quite recall.

File Description Example
TopicChrome.spark Layout for all the topic pages The default topic chrome from FubuDocs
SplashChrome.spark Layout for all the splash pages The default splash chrome from FubuDocs
NavigationBar.spark The layout of the top navigation bar that will generally appear on almost all pages The default top navigation from FubuDocs

Layout Helpers

In order to make the skinning easier without having to learn FubuMVC and Spark, we've endeavored to add custom html tags for all of the common elements of a FubuDocs layout.

  • <MailingList text="some text"/> - Creates a link to the mailing list from the project.xml file with the given text.

  • <SocialIcons /> - Creates a link with icons to the GitHub and Twitter pages defined in the project.xml file if they exist.

  • <RootLink text="some text" /> - Creates a link to the home page of the doc application. Using this is important to enable the various export modes described in Exporting to Static Html

  • <ProjectLogo /> - Creates a link to the root page of the current project

  • <TopicTitle /> - Just the title of the current topic, used like <h1><TopicTitle /></h1>

  • <ProjectName /> - Just the name of the current project, used like <h1><ProjectName /></h1>

  • <CurrentVersion /> - Just the first three digits of the semantic version of the library or application. Auto-magically determined by trying to find an assembly matching the documentation project name and reading the assembly version. If the documentation project is FubuCore.Docs, then FubuDocs looks for FubuCore.dll.

  • <commonHeader /> - Non visual helper that must be in your markup's <head> in order for the necessary CSS and JS assets to be loaded.

  • <commonFooter /> - Small helper that should be at the bottom of your markup's <body> tag in order for the necessary JS scripts to be loaded.

  • <InnerContent /> - For splash and topic pages, this binding controls is where the topic content will be rendered within the chrome pages.

  • <AuthoringTopic /> - Renders the authoring section for links to edit the existing page and access to the project tools in development mode. In the left pane of the default layout.

  • <TopTopicNavigation /> - Renders the very top row of navigation links. In this site, this content includes links to the home page, the current project, and social media links.

  • <LeftTopicNavigation /> - Renders the navigation links for the next and previous topics that appear in the left pane of this site.

  • <TopTopicNavigation /> - Renders the navigation links for the next and previous topics that appear in the 2nd to the top row of this site.
