Smoke Testing a Documentation Project

fubudocs smokeTests each documentation endpoint for Http errors for the doc projects in this folder

The smoke command will run your FubuDocs application and try to exercise each endpoint in the system looking for an HTTP 200 status code. Use the smoke command just to do a sanity check on your content.
fubudocs smoke[-n, --no-bottling] [-w, --watched] [-b, --browser Chrome|Firefox] [-b, --build ] [-o, --open] [-h, --host ]
Flags
[-n, --no-bottling]Disables the bottle and code snippet scanning while this command runs
[-w, --watched]Open a 'watched' browser with WebDriver to refresh the page on content or application recycles
[-b, --browser Chrome|Firefox]Browser to use in watched. Default is Firefox because it's more stable.
[-b, --build <build>]If you are running a class library, sets the preference for the profile to load. As in bin/[BuildFlag]. Default is debug
[-o, --open]Start the default browser to the home page of this application
[-h, --host <host>]Specify the directory for a hosting application for your documentation. Will be loaded as a FubuMVC Bottle
