Topics

FubuDocs was heavily influenced by the navigation and organization structure from readthedocs.org. As such, we had a couple goals in mind:

  1. Make it easy and low ceremony to author content
  2. Provide a sensible and maintainable way to establish navigation and organizational structure
  3. Run the resulting website hosted in FubuMVC;-)

With those goals in mind, FubuDocs uses the Spark View Engine support within LINK TO PROJECT 'FubuMVC' to generate content to utilize its out of the box support for authoring html content with Markdown and Spark's very powerful custom html bindings.

To author content with FubuDocs you simply write content into a spark text file either using Markdown or straight html. In FubuDocs nomemclature, each spark file is a "topic."

FubuDocs itself handles the basic page layout of each topic and renders standard navigation headers, footers, and a left column table of contents for each page based on the file structure of the topic files within the documentation project and the <section> structure of the rendered topic html.

FubuDocs uses Twitter Bootstrap for styling and layout -- but I betcha knew that from a single glance at this page.

A Topic File

A topic file is just a text file with the *.spark file extension. The structure of a topic file is going to follow the basic pattern shown below:

<!--Title: Some title-->
<markdown>

Content here....

</markdown>

The title of the topic is embedded as an HTML comment in the pattern <!--Title: [the title]--> anywhere in the file. Using this well known comment structure rather than an html element allows FubuDocs to reuse the title for a topic in navigation structures, the actual <title> element on the page, and in the topic "chrome" that FubuDocs wraps around the topic content.

The <markdown> element is not mandatory, but you will probably want to use markdown over html. The custom Spark bindings in FubuDocs are mostly usable from within markdown markup as well.

Sections within a Topic

To break up a longer topic, you can use the <Section> custom element to provide some structure within a topic and FubuDocs will build navigation for you in the left column.

Topic Url's

The url of a topic file is derived from the name of the file, the name of the documentation project, and the folder structure within the documentation project. Here are some examples from FubuDocs's own documentation:
Relative File Path Url
3.topics.spark /fubudocs/topics
6.commands/1.snippets.spark /fubudocs/commands/snippets
6.commands/index.spark /fubudocs/commands
splash.spark /fubudocs
index.spark /fubudocs/index
A couple things to note: * _index.spark_ has a special meaning as the main topic underneath a folder. * The number prefix you use to force the ordering of both folders and topics within a folder is ignored for the url determination * If there is a splash page, the file index.spark will be _[project name]/index_. * The sub folder names are part of the topic url You can also override the file specific, last part of the url for a topic by embedding <!-- Url: [the url]--> anywhere in the topic spark file.

Ordering Topics

Topic files are ordered within a folder by these rules:

  1. Any file named index.spark is first
  2. If files are prefixed with a number like 3.topics.spark and 2.splashpages.spark, the numbers are used to order
  3. Without any numeric prefixes the files are ordered alphabetically by the file name
  4. Child folders and files are ordered together

Authoring Toolbar

When running in development mode, you'll see an authoring toolbar in the left pane of the page for the current topic and documentation project. The file name is a link to open the current topic in a text editor. The other link leads to the project page described in the next topic.
